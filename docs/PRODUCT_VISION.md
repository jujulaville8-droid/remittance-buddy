# Remittance Buddy — Product Vision

## Core Identity

**We help immigrants and overseas workers keep more of their money.**

Not a comparison site. A decision engine that's on the user's side.

## Design Principles

1. **Trust over cleverness** — transparent math, plain language, visible tradeoffs
2. **Decide, not just display** — rank intelligently, explain why
3. **Every screen reduces friction** — shortest path from open to recommendation
4. **Design for repeat usage** — remember preferences, build financial habits
5. **Never fake certainty** — if data is incomplete, say so clearly

## Priority Corridor

**US → Philippines** (then Canada → Philippines)

Optimize everything for this community first: language, providers, payout methods, common amounts, trust signals, GCash/Maya support.

## Recommendation Categories

- Best overall
- Cheapest (lowest total cost to sender)
- Best for recipient (most pesos received)
- Fastest delivery
- Best for cash pickup
- Most trusted brand

## Must-Build Features (Phased)

### Phase 1 — Comparison + Recommendations + Affiliate
- Intelligent scoring engine with weighted factors
- Recommendation explanations ("why this won")
- Savings visibility ("you save $8.40", "recipient gets ₱620 more")
- Smart AI that asks only necessary questions
- Corridor specialization (US → PH)

### Phase 2 — Accounts + Memory + Alerts
- Repeat-user memory (corridor, amount, payout method, provider preference)
- Rate alerts ("PHP rate improved today")
- Saved corridors and preferences
- "Ready to send again?" prefilled experience

### Phase 3 — Profiles + Analytics
- Recipient profiles
- Transfer history and trends
- Provider profile pages (SEO + education)
- Historical best-option tracking

### Phase 4 — Embedded Transfers
- Direct provider routing partnerships
- Embedded transfer execution

### Phase 5 — Diaspora Financial Platform
- Bill pay abroad
- Tuition support
- Emergency sending
- Family support planning
- Recurring household remittance tools

## Moats

1. **Data** — anonymized comparison behavior improves recommendations
2. **Personalization** — product gets better with each use
3. **Content** — structured SEO pages from real comparison data
4. **Brand** — emotional identity with diaspora community

## Hard Rules

- Never build a generic table and call it done
- Never use financial jargon without translating it
- Never overload with too many inputs at once
- Never recommend without explaining reasoning
- Never hide uncertainty
- Never make AI overly verbose or gimmicky
- Never prioritize visual flash over trust and clarity
